# Engram MCP Benchmarks

## Running benches

Run all bench suites:

```sh
cargo bench
```

Run a single bench binary:

```sh
cargo bench --bench scoring_bench
```

## Baseline workflow

Save a baseline before making changes:

```sh
cargo bench -- --save-baseline before
```

Make your changes, then compare against the baseline:

```sh
cargo bench -- --baseline before
```

Criterion prints a regression/improvement summary for each bench case.

## HTML reports

After each run, criterion generates HTML reports under `target/criterion/`.
Open `target/criterion/report/index.html` in a browser to browse all results
with time-series plots.

## Bench isolation

Each bench binary instantiates its own `EmbeddingService` via
`std::sync::OnceLock` (see `benches/common/mod.rs::embedding_service`).
Model load happens once per binary, outside of any measured iteration.
Criterion runs iterations single-threaded by default, so there is no
contention on the model mutex during measurement.

## Two complementary harnesses

| Harness | Location | What it measures |
|---------|----------|-----------------|
| criterion | `benches/*.rs` | Latency / throughput of hot paths (scoring, embedding, SQL query plans) |
| cargo test | `tests/retrieval_quality.rs` | Recall quality: Hit@1, Hit@3, Hit@5, MRR |

These two concerns are intentionally separated. The criterion benches give you
fast feedback on performance regressions without needing to run the full
retrieval quality scorecard. The `retrieval_quality.rs` test uses real
embeddings and a realistic corpus to verify that changes do not hurt recall.

## Synthetic vs. real embeddings in benches

The 1k and 10k corpus benches (`query_bench`, `context_bench`) use synthetic
(dummy) vectors generated by a deterministic formula. This makes seeding fast
enough to run 10 000-memory corpora in CI-friendly time, but cosine similarity
scores are meaningless against a real query embedding.

These benches measure SQL retrieval cost, candidate pre-filtering, and
ranking pipeline throughput, NOT semantic recall quality. For recall, run:

```sh
cargo test --test retrieval_quality -- --nocapture
```
